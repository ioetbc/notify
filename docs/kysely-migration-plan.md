# Plan: Migrate from Raw SQL to Kysely

## Overview
Convert `/server/functions/admin/index.ts` from using raw SQL via `@neondatabase/serverless` to using Kysely, a type-safe SQL query builder with auto-generated types.

## Why Kysely over Knex
- Type-safe queries with full TypeScript IntelliSense
- `kysely-codegen` generates types directly from your database
- No manual type definitions needed
- Modern, actively maintained

## Files to Modify/Create

| File | Action |
|------|--------|
| `server/package.json` | Add `kysely`, `pg`, `kysely-codegen` dependencies |
| `server/db/kysely.ts` | Create - Kysely configuration and connection |
| `server/db/types.ts` | Auto-generated by `kysely-codegen` |
| `server/db/index.ts` | Create - Re-exports |
| `server/functions/admin/index.ts` | Modify - Convert all 30+ SQL queries to Kysely |

## Implementation Steps

### Step 1: Install Dependencies
```bash
cd server && bun add kysely pg && bun add -D kysely-codegen @types/pg
```

### Step 2: Generate Types from Database
```bash
# Add to package.json scripts
"db:types": "kysely-codegen --out-file db/types.ts"

# Run with connection string
DATABASE_URL="your-neon-connection-string" bun db:types
```

This introspects your Neon database and generates `db/types.ts` with interfaces for all tables.

### Step 3: Create Database Module

**`server/db/kysely.ts`**:
```typescript
import { Kysely, PostgresDialect } from 'kysely';
import { Pool } from 'pg';
import { Resource } from 'sst';
import type { DB } from './types';

let dbInstance: Kysely<DB> | null = null;

export function db(): Kysely<DB> {
  if (!dbInstance) {
    dbInstance = new Kysely<DB>({
      dialect: new PostgresDialect({
        pool: new Pool({
          connectionString: Resource.NeonDB.connectionString,
          ssl: { rejectUnauthorized: false },
          max: 1,
        }),
      }),
    });
  }
  return dbInstance;
}
```

### Step 4: Convert Queries in admin/index.ts

Replace import:
```typescript
// Remove: import { neon } from '@neondatabase/serverless';
// Remove: const sql = neon(Resource.NeonDB.connectionString);
import { db } from '../../db';
```

**Query Conversion Examples:**

| Query Type | Before (Raw SQL) | After (Kysely) |
|------------|------------------|----------------|
| SELECT all | `sql\`SELECT * FROM customer\`` | `db().selectFrom('customer').selectAll().execute()` |
| SELECT where | `sql\`SELECT * FROM workflow WHERE id = ${id}\`` | `db().selectFrom('workflow').selectAll().where('id', '=', id).executeTakeFirst()` |
| SELECT with JOIN | See complex query below | Chain `.innerJoin()` / `.leftJoin()` |
| INSERT returning | `sql\`INSERT INTO customer (...) RETURNING *\`` | `db().insertInto('customer').values({...}).returningAll().executeTakeFirstOrThrow()` |
| UPDATE returning | `sql\`UPDATE workflow SET ... RETURNING *\`` | `db().updateTable('workflow').set({...}).where('id', '=', id).returningAll().executeTakeFirstOrThrow()` |
| DELETE returning | `sql\`DELETE FROM workflow WHERE ... RETURNING *\`` | `db().deleteFrom('workflow').where('id', '=', id).returningAll().executeTakeFirst()` |
| Raw SQL (enums) | `WHERE enumtypid = 'trigger_event'::regtype` | `db().selectFrom('pg_enum').select('enumlabel as value').where(sql\`enumtypid = 'trigger_event'::regtype\`).execute()` |

**Complex JOIN Example:**
```typescript
const steps = await db()

    'ss.body as send_body',
    'ss.next_step_id as send_next_step_id',

```

### Route 4: POST `/workflows` (line 77-179) - Use Transaction
```typescript
// After - Wrap everything in transaction
const result = await db().transaction().execute(async (trx) => {
  // Get or create customer
  let customerId = body.customer_id;
  if (!customerId) {
    const existing = await trx
      .selectFrom('customer')
      .select('id')
      .limit(1)
      .executeTakeFirst();

    if (existing) {
      customerId = existing.id;
    } else {
      const newCustomer = await trx
        .insertInto('customer')
        .values({ email: 'dev@example.com', name: 'Dev Customer' })
        .returning('id')
        .executeTakeFirstOrThrow();
      customerId = newCustomer.id;
    }
  }

  // Create workflow
  const workflow = await trx
    .insertInto('workflow')
    .values({
      customer_id: customerId,
      name: body.name,
      trigger_event: body.trigger_event,
      active: true,
    })
    .returningAll()
    .executeTakeFirstOrThrow();

  // Insert steps
  const idMap = new Map<string, string>();
  for (const step of body.steps) {
    const dbStep = await trx
      .insertInto('step')
      .values({ workflow_id: workflow.id, step_type: step.type })
      .returning('id')
      .executeTakeFirstOrThrow();

    idMap.set(step.id, dbStep.id);

    // Insert step config based on type
    if (step.type === 'wait') {
      await trx
        .insertInto('step_wait')
        .values({ step_id: dbStep.id, hours: step.config.hours || 24 })
        .execute();
    } else if (step.type === 'branch') {
      await trx
        .insertInto('step_branch')
        .values({
          step_id: dbStep.id,
          user_column: step.config.user_column || 'unconfigured',
          operator: step.config.operator || '=',
          compare_value: step.config.compare_value || null,
        })
        .execute();
    } else if (step.type === 'send') {
      await trx
        .insertInto('step_send')
        .values({
          step_id: dbStep.id,
          title: step.config.title || 'Notification',
          body: step.config.body || '',
        })
        .execute();
    }
  }

  // Update edges
  for (const edge of body.edges) {
    const sourceDbId = idMap.get(edge.source);
    const targetDbId = idMap.get(edge.target);
    if (!sourceDbId || !targetDbId) continue;

    const sourceStep = body.steps.find((s) => s.id === edge.source);
    if (!sourceStep) continue;

    if (sourceStep.type === 'wait') {
      await trx
        .updateTable('step_wait')
        .set({ next_step_id: targetDbId })
        .where('step_id', '=', sourceDbId)
        .execute();
    } else if (sourceStep.type === 'send') {
      await trx
        .updateTable('step_send')
        .set({ next_step_id: targetDbId })
        .where('step_id', '=', sourceDbId)
        .execute();
    } else if (sourceStep.type === 'branch') {
      if (edge.sourceHandle === 'yes') {
        await trx
          .updateTable('step_branch')
          .set({ true_step_id: targetDbId })
          .where('step_id', '=', sourceDbId)
          .execute();
      } else if (edge.sourceHandle === 'no') {
        await trx
          .updateTable('step_branch')
          .set({ false_step_id: targetDbId })
          .where('step_id', '=', sourceDbId)
          .execute();
      }
    }
  }

  return { workflow, idMap: Object.fromEntries(idMap) };
});
```

## Migration Workflow

1. **Install dependencies**
   ```bash
   cd server && bun add kysely pg && bun add -D kysely-codegen @types/pg
   ```

2. **Add db:types script to package.json**
   ```json
   "db:types": "kysely-codegen --out-file db/types.ts"
   ```

3. **Generate types** (requires DATABASE_URL env var)
   ```bash
   DATABASE_URL="postgresql://..." bun db:types
   ```

4. **Create `server/db/` directory**
   - `kysely.ts` - connection singleton
   - `index.ts` - re-exports db function

5. **Migrate `admin/index.ts`**
   - Update imports
   - Convert routes one at a time
   - Test after each conversion

## Serverless Considerations

- Pool `max: 1` prevents connection exhaustion
- Singleton pattern reuses connection across warm Lambda invocations
- SSL enabled for Neon
